# Задание 1

## Задача 1

### Текст задачи

Разработать консольное приложение с дружественным интерфейсом с возможностью выбора заданий для работы с «базой данных (БД)», хранящейся в бинарном файле. Перечень полей (минимум 5), достаточно полно характеризующих заданную в варианте предметную область, предложить самостоятельно (постараться отразить в перечне полей такие, которые требуют разных типов данных). Приложение должно выполнять следующие функции: 
1. Чтение базы данных из бинарного файла. 
2. Просмотр базы данных. 
3. Удаление элементов (по ключу). 
4. Добавление элементов. 
5. Реализация 4 запросов (формулировки запросов придумать самостоятельно). 2 запроса должны возвращать перечень, 2 запроса одно значение. 
В классе должны присутствовать свойства, конструкторы, перегруженный метод ToString(). Весь функционал приложения реализовать в виде методов вспомогательного класса с помощью LINQ запросов. 
Предусмотреть обработку возможных ошибок при работе программы.

### Алгоритм решения

Программа состоит из четырёх классов.

Dish — класс блюда. Содержит поля: Id (автоинкремент), Name, Price, Calories, Spiciness, IsAvailable. Перегруженный ToString() выводит все поля в табличном виде с фиксированной шириной колонок. Метод GetShortInfo() возвращает только название.

CafeManager — вспомогательный класс с основной логикой. Хранит список блюд и путь к файлу. LoadFromFile() и SaveToFile() читают и записывают бинарный файл через BinaryReader/BinaryWriter (формат: количество блюд, затем для каждого Id, Name, Price, Calories, Spiciness, IsAvailable). Методы ShowShortMenu(), ShowFullMenu(), ShowSingleFullMenu() отвечают за вывод. AddDishWithParams(), DeleteDishByIndex(), EditDishByIndex() — добавление, удаление, редактирование. Все изменения сразу сохраняются в файл.

InputValidator — проверка ввода. Даёт 3 попытки на каждое действие, после чего операция отменяется. Проверяет: название не пустое, цена > 0, калории ≥ 0, острота 0–10, наличие как true/false или 1/0. При запуске проверяет наличие файла, при отсутствии предлагает создать (тоже 3 попытки).

Program — главный класс. В Main после проверки файла запускает цикл с меню действий (1–7, 0 — выход). Все действия вынесены в отдельные методы (например, AddDishAction(), EditDishAction()), внутри которых вызываются методы CafeManager и InputValidator. Отдельное подменю «Анализ меню» вызывает 4 LINQ-запроса.

LINQ-запросы (в классе CafeManager):

Блюда в наличии, по алфавиту (перечень) — _menu.Where(d => d.IsAvailable).OrderBy(d => d.Name).ToList().

Блюда с калорийностью выше средней (перечень) — сначала _menu.Average(d => d.Calories), затем _menu.Where(d => d.Calories > averageCalories).ToList().

Средняя цена (одно значение) — _menu.Average(d => d.Price).

Самое острое блюдо (одно значение) — _menu.Max(d => d.Spiciness), затем _menu.Where(d => d.Spiciness == maxSpiciness).ToList(). Если максимум 0 — возвращает пустой список (нет острых блюд).

Результаты запросов выводятся в подменю анализа
